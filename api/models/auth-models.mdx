## Overview

The authentication models handle login, token refresh, and session management for different platforms (web and mobile).

## Payload Models

### LoginPayload

Payload for login requests. Requires either `email` or `phoneNumber`, along with `password`.

**Source:** `src/app/core/models/auth/auth.payload.ts:8`

```typescript
interface LoginPayload {
  email?: string;
  phoneNumber?: string;
  password: string;
  location?: Location;
  appAudience: AppAudience;
  expectedUserType: UserType;
}
```

<ResponseField name="email" type="string" optional>
  User's email address. Either `email` or `phoneNumber` is required.
</ResponseField>

<ResponseField name="phoneNumber" type="string" optional>
  User's phone number. Either `email` or `phoneNumber` is required.
</ResponseField>

<ResponseField name="password" type="string" required>
  User's password for authentication.
</ResponseField>

<ResponseField name="location" type="Location" optional>
  Geographic location of the user at login time.
  
  <Expandable title="Location properties">
    <ResponseField name="latitude" type="number" required>
      Latitude coordinate
    </ResponseField>
    <ResponseField name="longitude" type="number" required>
      Longitude coordinate
    </ResponseField>
    <ResponseField name="city" type="string" optional>
      City name
    </ResponseField>
    <ResponseField name="country" type="string" optional>
      Country name
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="appAudience" type="AppAudience" required>
  Identifies which application is making the request.
  
  **Possible values:** `driver_app` | `passenger_app` | `admin_panel` | `api_client`
</ResponseField>

<ResponseField name="expectedUserType" type="UserType" required>
  The expected user type for this login attempt.
  
  **Enum values:** `passenger` | `driver` | `admin`
</ResponseField>

<CodeGroup>
```json Example Request
{
  "email": "driver@example.com",
  "password": "SecurePassword123!",
  "location": {
    "latitude": 40.7128,
    "longitude": -74.0060,
    "city": "New York",
    "country": "USA"
  },
  "appAudience": "driver_app",
  "expectedUserType": "driver"
}
```
</CodeGroup>

## Response Models

### BaseAuthResponse

Common fields returned by the backend in login/refresh responses.

**Source:** `src/app/core/models/auth/auth.response.ts:4`

```typescript
interface BaseAuthResponse {
  accessToken: string;
  accessTokenExpiresAt: number;
  refreshTokenExpiresAt?: number;
  sid?: string;
  sessionType: SessionType;
}
```

<ResponseField name="accessToken" type="string" required>
  JWT access token for authenticating API requests.
</ResponseField>

<ResponseField name="accessTokenExpiresAt" type="number" required>
  Expiration timestamp for the access token in epoch milliseconds.
</ResponseField>

<ResponseField name="refreshTokenExpiresAt" type="number" optional>
  Expiration timestamp for the refresh token in epoch milliseconds. Present in login/refresh responses.
</ResponseField>

<ResponseField name="sid" type="string" optional>
  Session ID (JTI). Primarily present in refresh responses.
</ResponseField>

<ResponseField name="sessionType" type="SessionType" required>
  Type of session being created.
  
  **Enum values:** `web` | `mobile_app` | `admin_panel` | `api_client`
</ResponseField>

#### LoginResponseWeb

Login response for web applications. Refresh token is sent via HTTP-only cookie, not in response body.

**Source:** `src/app/core/models/auth/auth.response.ts:18`

```typescript
interface LoginResponseWeb extends BaseAuthResponse {
  refreshToken?: never;
}
```

<ResponseField name="refreshToken" type="never">
  Explicitly absent from response body. Refresh token is set as HTTP-only cookie for web clients.
</ResponseField>

<ResponseField name="...BaseAuthResponse" type="BaseAuthResponse">
  All properties from BaseAuthResponse are included.
</ResponseField>
